Hexo搭建的坑及常用命令

发表于 2020-03-12 |

花了接近四个小时终于部署好了Hexo……总体来説部署难度不高但是细节很多，有几个易错点我反反复复操作了五六次才解决(吐血.jpg)

0x00
关于hexo d部署报错(配置问题)的解决方案

$ hexo d
You should configure deployment settings in _config.yml first!

Available deployer plugins:

For more help, you can check the online docs: https://hexo.io/

如上报错在_config.yml中deploy设置是否首行缩进两格
rules:变量名之前两个空格，参数值前一个空格，用:分割

0x01
关于hexo d部署错误(模块缺失)解决方案

ERROR Deployer not found: git

如上是因为没安装hexo-deployer-git插件，在站点目录下输入下面的插件安装就好了npm install hexo-deployer-git –save

0x02

搭建完成后乱码问题解决

在source目录下新建一个.md文档并添加如下内容
:set encoding=utf-8
:set fileencodings=ucs-bom,utf-8,cp936
:set fileencoding=gb2312
:Set termencoding=utf-8

hexo清除缓存十分重要：hexo clean

$ hexo new “postName” #新建文章
$ hexo new page “pageName” #新建页面
$ hexo generate (或者hexo g) 生成静态文件，会在当前目录下生成一个新的叫做public的文件夹
$ hexo server (或者hexo s) 启动本地web服务，用于博客的预览
$ hexo deploy (或者hexo d) 部署播客到远端（比如github, coding等平台）
常用简写：

$ hexo n == hexo new

$ hexo g == hexo generate

$ hexo s == hexo server

$ hexo d == hexo deploy

$ hexo clean #清除
$ hexo g #生成
$ hexo d #部署 # 可与hexo g合并为 hexo d -g
$ hexo s -g #生成预览
